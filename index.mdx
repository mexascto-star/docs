The **Mex Pet API** is a REST API that lets you automate everything your ranch does inside the app: manage contacts and their pets, create and update bookings, read your service catalog, send SMS or email to customers, and more.

It is designed to feel familiar if you have used Stripe, GoHighLevel, or similar platforms:

- **Predictable, resource-oriented URLs** — `/contacts`, `/pets`, `/bookings`, `/services`, `/messages`
- **JSON everywhere** — all request and response bodies are JSON
- **Standard HTTP verbs** — `GET`, `POST`, `PATCH`, `DELETE`
- **Bearer-token auth with scopes** — every token is bound to one business and only the permissions you grant

## Base URL

All requests go to a single base URL. Append the resource path to it.

```text
https://umyxbfjlngqhcminmwgq.supabase.co/functions/v1/api-v1
```

For example, listing contacts is:

```text
GET https://umyxbfjlngqhcminmwgq.supabase.co/functions/v1/api-v1/contacts
```

## Tenancy & isolation

Every token belongs to exactly one business (your ranch). The API automatically scopes every request to that business — you can never read or write another business's data, even by guessing IDs. There is nothing extra to configure.

## Versioning

The current version is **v1**, encoded in the path (`/api-v1`). Breaking changes ship under a new version; additive changes (new fields, new endpoints) may appear within v1 without notice, so write your integration to ignore unknown fields.
