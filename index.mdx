The **Mex Pet API** lets you build on top of everything your ranch does inside the app — programmatically. Sync contacts and their pets, create and update bookings, read payments, manage your team, fire messages, and drive automations, all over a clean REST interface.

It's designed to feel familiar if you've used **Stripe** or **GoHighLevel**:

## Base URL

Every request goes to a single base URL; append the resource path.

```text
https://mex-pet-app.vercel.app/api/v1
```

<Info>
  This base URL is served by your app and proxied to the API. When you move to a production domain, only this URL changes — your integration code keeps working.
</Info>

For example, listing contacts is `GET https://mex-pet-app.vercel.app/api/v1/contacts`.

## Tenancy & isolation

Every token belongs to exactly one business (your ranch). The API automatically scopes every request to that business — you can never read or write another business's data, even by guessing IDs. There's nothing extra to configure.

## Versioning

The current version is **v1**, encoded in the path (`/api/v1`). Breaking changes ship under a new version; additive changes (new fields, new endpoints) can appear within v1 without notice — so write your integration to ignore unknown fields.
