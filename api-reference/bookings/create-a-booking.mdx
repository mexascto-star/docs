---
title: "Create a booking"
description: "Creates a new booking reservation. Requires the bookings.write scope."
openapi: "POST /bookings"
---

Creates a new reservation. A booking must be linked to an existing contact — it cannot exist without a valid customer.

**Required scope:** `bookings.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

<Tip>
  Use an `Idempotency-Key` header so that network retries never create duplicate bookings.
</Tip>

## Request

```bash
POST /bookings
Authorization: Bearer mxp_live_your_token_here
Content-Type: application/json
```

## Body fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `contact_id` | string | ✅ | ID of the customer. Must be an existing contact in your business. |
| `check_in_date` | date | ✅ | Check-in date (YYYY-MM-DD) |
| `check_out_date` | date | — | Check-out date (YYYY-MM-DD) |
| `pet_id` | string | — | Optional pet in your business. Linked to the booking via `pet_ids` and the `booking_pets` junction. |
| `service_type` | string | — | `boarding`, `grooming`, `daycare`, `training` |
| `sub_services` | array | — | Array of line-item objects (see format below) |
| `total_amount` | number | — | Total charge in dollars |
| `payment_status` | string | — | Initial payment status (default `unpaid`) |
| `has_pickup` | boolean | — | Whether pickup transport is needed |
| `has_dropoff` | boolean | — | Whether dropoff transport is needed |

<Warning>
  `contact_id` must be an existing contact in your business. An invalid ID returns `400`: _"contact\_id does not match a contact in your business."_

  **No `status` on create.** A new booking is **always** created as `pending`. Status is set later from the app — do not send a `status` field on create.
</Warning>

### `sub_services` format

Each item in the `sub_services` array is a line-item object:

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | Line-item name (e.g. `"Dog Boarding (5 x 5)"`) |
| `price` | number | Line price |
| `pet_id` | uuid \| null | Pet this line is for, or `null` for booking-level lines like transport |
| `quantity` | number | Quantity for this line |
| `service_category` | string | e.g. `Boarding`, `Transport`, `Grooming` |
| `address` | string | Optional — used for transport legs |

```json
[
  {
    "name": "Dog Boarding (5 x 5)",
    "price": 24,
    "pet_id": "f19a54f6-90d2-48a4-a39f-74db540f2213",
    "address": "Punjab, Bahawalpur, Government Housing Society",
    "quantity": 1,
    "service_category": "Boarding"
  },
  {
    "name": "Pick Up",
    "price": 50,
    "pet_id": null,
    "quantity": 1,
    "service_category": "Transport"
  }
]
```

## Example

```bash
curl -X POST https://mex-pet-app.vercel.app/api/v1/bookings \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: d4e5f6a7-b8c9-0123-def0-456789012345" \
  -d '{
    "contact_id": "usr_6f1c3d",
    "check_in_date": "2026-08-01",
    "check_out_date": "2026-08-05",
    "service_type": "boarding",
    "sub_services": [
      { "name": "Standard Boarding", "price": 55 },
      { "name": "Bath & Brush", "price": 25 }
    ],
    "total_amount": 245,
    "has_pickup": true,
    "has_dropoff": true
  }'
```
