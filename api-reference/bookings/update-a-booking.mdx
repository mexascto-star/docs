---
title: "Update a booking"
description: "Updates fields on a booking. Requires the bookings.write scope."
openapi: "PATCH /bookings/{id}"
---

Updates an existing booking. Use this to change status, reassign staff, record payment, or adjust dates and services.

**Required scope:** `bookings.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

## Request

```bash
PATCH /bookings/{id}
Authorization: Bearer mxp_live_your_token_here
Content-Type: application/json
```

## Updatable fields

| Field | Type | Description |
| --- | --- | --- |
| `service_type` | string | Primary service type |
| `sub_services` | array | Line items `[{ name, price }]` — replaces existing list |
| `check_in_date` | date | Check-in date (YYYY-MM-DD) |
| `check_out_date` | date | Check-out date (YYYY-MM-DD) |
| `total_amount` | number | Total charge |
| `payment_status` | string | `unpaid`, `partial`, `paid`, `refunded` |
| `assigned_staff_id` | string | Staff member ID to assign |
| `status` | string | `pending`, `in_progress`, `completed`, `cancelled` |

## Common one-liners

**Check a pet in:** `{ "status": "in_progress" }`

**Mark as paid:** `{ "payment_status": "paid" }`

**Assign staff:** `{ "assigned_staff_id": "stf_ee5f6a" }`

**Complete:** `{ "status": "completed", "payment_status": "paid" }`

## Example

```bash
curl -X PATCH https://mex-pet-app.vercel.app/api/v1/bookings/bkg_cc3d4e \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "in_progress",
    "assigned_staff_id": "stf_ee5f6a"
  }'
```
