---
title: "Update a staff member"
description: "Updates a staff member's profile. Requires the staff.write scope."
openapi: "PATCH /staff/{id}"
---

Updates a team member's profile. Note that changing `staff_type` is a role change — it updates the underlying user's system role as well.

**Required scope:** `staff.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

## Request

```bash
PATCH /staff/{id}
Authorization: Bearer mxp_live_your_token_here
Content-Type: application/json
```

## Updatable fields

| Field | Type | Description |
| --- | --- | --- |
| `full_name` | string | Display name |
| `phone` | string | Contact phone |
| `staff_type` | string | `worker`, `driver`, `receptionist` — also changes user role |
| `service_type` | string | Specialization |
| `is_active` | boolean | Set `false` to deactivate without deleting |

## Example — deactivate

```bash
curl -X PATCH https://mex-pet-app.vercel.app/api/v1/staff/stf_ff6a7b \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{ "is_active": false }'
```
