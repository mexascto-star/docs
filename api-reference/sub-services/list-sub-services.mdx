---
title: "List sub-services"
description: "Returns sub-services for a parent service. Requires the services.read scope."
---

Returns sub-services (add-ons and line items) from the `sub_services` table. Always filter by `service_id` to get the sub-services for a specific parent. Sub-services are also embedded in `GET /services/{id}` responses.

**Required scope:** `services.read`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

## Query parameters

| Parameter | Type | Description |
| --- | --- | --- |
| `limit` | integer | Page size (1–100, default `20`) |
| `offset` | integer | Number of items to skip |
| `service_id` | string | Filter to sub-services of this parent service |
| `active` | boolean | `true` = active sub-services only |

## Example

```bash
curl "https://mex-pet-app.vercel.app/api/v1/sub-services?service_id=svc_gg7b8c" \
  -H "Authorization: Bearer mxp_live_your_token_here"
```
