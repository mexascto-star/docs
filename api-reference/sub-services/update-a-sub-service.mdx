---
title: "Update a sub-service"
description: "Updates fields on a sub-service. Unset fields are left unchanged. Requires the services.write scope."
---

Updates one or more fields on an existing sub-service (PATCH semantics). The parent `service_id` cannot be changed after creation.

**Required scope:** `services.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

## Updatable fields

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | Sub-service name |
| `description` | string | Full description |
| `price` | number | Price in dollars |
| `duration_minutes` | integer | Estimated duration |
| `sort_order` | integer | Display order |
| `is_active` | boolean | Availability toggle |
| `pricing_model` | string | e.g. `fixed` |
| `unit_label` | string | Unit label |

## Example

```bash
curl -X PATCH https://mex-pet-app.vercel.app/api/v1/sub-services/sub_aa1b2c \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "price": 45,
    "is_active": false
  }'
```
