---
title: "Create a pet"
description: "Creates a pet record linked to a contact. Requires the pets.write scope."
openapi: "POST /pets"
---

Creates a pet and links it to an existing contact (owner). A pet cannot exist without a valid owner.

**Required scope:** `pets.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

## Request

```bash
POST /pets
Authorization: Bearer mxp_live_your_token_here
Content-Type: application/json
```

## Body fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `contact_id` | string | ✅ | ID of the owning contact. Must exist in your business. |
| `name` | string | ✅ | Pet's name |
| `type` | string | — | Animal type: `dog`, `cat`, `bird`, etc. |
| `breed` | string | — | Breed |
| `age` | string | — | Age (free-form: `"3"`, `"18 months"`) |
| `weight` | string | — | Weight (free-form: `"65 lbs"`) |
| `gender` | string | — | `male` or `female` |
| `allergies` | string | — | Known allergies |
| `health_notes` | string | — | Free-text health information |
| `is_vaccinated` | boolean | — | Whether vaccinations are current |
| `status` | string | — | `active` (default) or `inactive` |

<Warning>
  `contact_id` must be an existing contact in your business. An unknown ID returns `400`: _"contact\_id does not match a contact in your business."_
</Warning>

## Example

```bash
curl -X POST https://mex-pet-app.vercel.app/api/v1/pets \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: c3d4e5f6-a7b8-9012-cdef-345678901234" \
  -d '{
    "contact_id": "usr_6f1c3d",
    "name": "Max",
    "type": "dog",
    "breed": "Golden Retriever",
    "age": "2",
    "weight": "55 lbs",
    "gender": "male",
    "is_vaccinated": true
  }'
```
