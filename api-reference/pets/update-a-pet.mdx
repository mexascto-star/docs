---
title: "Update a pet"
description: "Updates fields on a pet record. Requires the pets.write scope."
openapi: "PATCH /pets/{id}"
---

Updates one or more fields on an existing pet (PATCH semantics). The pet's `contact_id` (owner) cannot be changed after creation.

**Required scope:** `pets.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

## Request

```bash
PATCH /pets/{id}
Authorization: Bearer mxp_live_your_token_here
Content-Type: application/json
```

## Updatable fields

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | Pet's name |
| `type` | string | Animal type |
| `breed` | string | Breed |
| `age` | string | Age (free-form) |
| `weight` | string | Weight (free-form) |
| `gender` | string | `male` or `female` |
| `allergies` | string | Known allergies |
| `health_notes` | string | Free-text health notes |
| `is_vaccinated` | boolean | Vaccination status |
| `status` | string | `active` or `inactive` |
| `custom_fields` | object | Business-defined custom values as a JSON object where **each key is a `field_key`** and the value is what you want to store. **Merged** into the existing values — keys you don't send are preserved. Unknown keys are ignored; `checkbox` values are coerced to booleans. See [Custom fields](/custom-fields). |

### `custom_fields` shape

Send only the keys you want to change — anything already stored under a key you omit is kept.

```json
"custom_fields": {
  "microchip_number": "985112000123456", // text
  "is_service_animal": true,             // checkbox → boolean
  "daily_meals": 3                       // number
}
```

## Example

Update weight, health notes, **and** two custom-field values in one call:

```bash
curl -X PATCH https://mex-pet-app.vercel.app/api/v1/pets/pet_aa1b2c \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "weight": "68 lbs",
    "health_notes": "Checkup completed July 2026 — all clear",
    "custom_fields": {
      "daily_meals": 3,
      "is_service_animal": false
    }
  }'
```
