---
title: "Create a contact"
description: "Creates a new contact. Requires the contacts.write scope."
openapi: "POST /contacts"
---

Creates a new customer in your ranch. If a user with the same email already exists on the platform, they are linked to your business — no duplicate is created.

**Required scope:** `contacts.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

<Tip>
  Send an `Idempotency-Key` header on this request so retries never create duplicate contacts.
</Tip>

## Request

```bash
POST /contacts
Authorization: Bearer mxp_live_your_token_here
Content-Type: application/json
```

## Body fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `full_name` | string | ✅ | Contact's display name |
| `email` | string | ✅ | Email address. If already exists, the account is linked. |
| `phone` | string | — | Phone number (E.164 recommended: `+15125550199`) |
| `address` | string | — | Street address |
| `city` | string | — | City |
| `state` | string | — | State or province |
| `zip_code` | string | — | Postal code |
| `country` | string | — | Country |
| `membership_tier` | string | — | e.g. `bronze`, `silver`, `gold` |
| `tags` | string\[\] | — | Tags to apply immediately |
| `custom_fields` | object | — | Business-defined custom values. Send as a JSON object where **each key is a `field_key`** from your business's custom-field registry and **the value is the value for that field** (see shape below). Unknown keys are ignored; `checkbox` values are coerced to booleans. Discover your available keys with [`GET /custom-fields?entity_type=customer`](/api-reference/custom-fields/list-custom-fields). |

### `custom_fields` shape

The `custom_fields` value is a plain JSON object of **key → value** pairs. The key is the `field_key` of a definition in your registry; the value is what you want to store.

```json
"custom_fields": {
  "favorite_toy": "Rope",       // text
  "vip": true,                  // checkbox → boolean
  "referral_source": "Instagram", // dropdown
  "lifetime_visits": 12,        // number
  "birthday": "2020-05-14"      // date (YYYY-MM-DD)
}
```

## Example

```bash
curl -X POST https://mex-pet-app.vercel.app/api/v1/contacts \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: a1b2c3d4-e5f6-7890-abcd-ef1234567890" \
  -d '{
    "full_name": "John Smith",
    "email": "john@example.com",
    "phone": "+15125550199",
    "city": "Austin",
    "state": "TX",
    "membership_tier": "bronze",
    "tags": ["new-client"],
    "custom_fields": {
      "favorite_toy": "Rope",
      "vip": true,
      "referral_source": "Instagram"
    }
  }'
```

## Error responses

| Status | Code | Meaning |
| --- | --- | --- |
| `400` | `invalid_request_error` | Missing required field (`full_name` or `email`) |
| `401` | `authentication_error` | Missing or invalid token |
| `403` | `permission_error` | Token missing `contacts.write` scope |
