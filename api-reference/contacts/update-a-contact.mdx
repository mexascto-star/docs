---
title: "Update a contact"
description: "Updates the supplied fields on a contact. Requires the contacts.write scope."
openapi: "PATCH /contacts/{id}"
---

Updates one or more fields on an existing contact. Only the fields you send are changed — all others remain as-is (PATCH semantics).

**Required scope:** `contacts.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

## Request

```bash
PATCH /contacts/{id}
Authorization: Bearer mxp_live_your_token_here
Content-Type: application/json
```

## Updatable fields

| Field | Type | Description |
| --- | --- | --- |
| `full_name` | string | Display name |
| `phone` | string | Phone number |
| `address` | string | Street address |
| `city` | string | City |
| `state` | string | State or province |
| `zip_code` | string | Postal code |
| `country` | string | Country |
| `membership_tier` | string | e.g. `bronze`, `silver`, `gold` |
| `tags` | string\[\] | **Replaces** the entire tag list |
| `avatar_url` | string | URL to profile photo |
| `custom_fields` | object | Map of custom values keyed by `field_key`. **Merged** into the existing values — keys you don't send are preserved. Unknown keys are ignored; `checkbox` values are coerced to booleans. See [Custom fields](/custom-fields). |

<Warning>
  `tags` is a **replace** operation. The array you send becomes the contact's complete tag list. To add one tag without removing others, read the current `tags` first and include them all in your PATCH.
</Warning>

<Note>
  `email`, user role, and business assignment cannot be changed via the API.
</Note>

## Example

```bash
curl -X PATCH https://mex-pet-app.vercel.app/api/v1/contacts/usr_9a2b4c \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "membership_tier": "silver",
    "tags": ["new-client", "referral"]
  }'
```
