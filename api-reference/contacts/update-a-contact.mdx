---
title: "Update a contact"
description: "Updates the supplied fields on a contact. Requires the contacts.write scope."
openapi: "PATCH /contacts/{id}"
---

Updates one or more fields on an existing contact. Only the fields you supply are changed — all others remain as-is. This is a partial update (PATCH semantics).

**Required scope:** `contacts.write`

## Updatable fields

| Field | Type | Description |
| --- | --- | --- |
| `full_name` | string | Display name |
| `phone` | string | Phone number |
| `address` | string | Street address |
| `city` | string | City |
| `state` | string | State or province |
| `zip_code` | string | Postal code |
| `country` | string | Country |
| `membership_tier` | string | e.g. `bronze`, `silver`, `gold` |
| `tags` | string\[\] | **Replaces** the full tag list |
| `avatar_url` | string | URL to profile photo |

<Warning>
  `tags` is a **replace** operation — the array you send becomes the contact's complete tag list. To add a single tag without losing others, read the current tags first and include them in your PATCH.
</Warning>

<Note>
  `email`, `role`, and business assignment cannot be changed via the API.
</Note>

## Example

```bash
curl -X PATCH https://mex-pet-app.vercel.app/api/v1/contacts/usr_9a2b4c \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "membership_tier": "silver",
    "tags": ["new-client", "referral"]
  }'
```

```json
{
  "id": "usr_9a2b4c",
  "object": "contact",
  "full_name": "John Smith",
  "email": "john@example.com",
  "phone": "+15125550199",
  "membership_tier": "silver",
  "tags": ["new-client", "referral"],
  "loyalty_points": 0,
  "avatar_url": null,
  "created_at": "2026-07-09T14:22:00.000Z"
}
```
