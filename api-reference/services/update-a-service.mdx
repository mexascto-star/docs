---
title: "Update a service"
description: "Updates a service in your catalog. Requires the services.write scope."
openapi: "PATCH /services/{id}"
---

Updates one or more fields on a service. Common uses: toggling `is_active`, adjusting pricing, updating the description, or linking/unlinking a `parent_id`.

**Required scope:** `services.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

<Note>
  When you update `title`, the API writes it to both the `name` and `title` columns so the name stays in sync in the app.
</Note>

## Request

```bash
PATCH /services/{id}
Authorization: Bearer mxp_live_your_token_here
Content-Type: application/json
```

## Updatable fields

| Field | Type | Description |
| --- | --- | --- |
| `title` | string | Service name — updates both `name` and `title` columns |
| `description` | string | Full description |
| `category` | string | `boarding`, `grooming`, `daycare`, `training`, `addon`, `retail` |
| `base_price` | number | Price in dollars |
| `price_type` | string | `nightly`, `fixed`, `tiered` |
| `price_unit` | string | `per_night`, `per_mile`, `fixed` |
| `is_active` | boolean | Toggle availability |
| `is_package` | boolean | Whether this is a package |
| `parent_id` | string | Set or clear the parent package link |

## Example — raise the price

```bash
curl -X PATCH https://mex-pet-app.vercel.app/api/v1/services/svc_ff6a7b \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{ "base_price": 60 }'
```

## Example — deactivate a service

```bash
curl -X PATCH https://mex-pet-app.vercel.app/api/v1/services/svc_ff6a7b \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{ "is_active": false }'
```
