---
title: "Create a service"
description: "Creates a new service in your catalog. Requires the services.write scope."
openapi: "POST /services"
---

Adds a new service to your catalog. Set `is_package: true` to mark it as a package. To add add-ons or line items, create them via `POST /sub-services` after creating the service.

**Required scope:** `services.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

<Note>
  The API writes `title` to both the `name` and `title` columns so the service name shows correctly in the Mex Pet app (which reads `name`).
</Note>

## Body fields

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `title` | string | ✅ | Service name — written to both `name` and `title` columns |
| `description` | string | — | Full description |
| `category` | string | — | `boarding`, `grooming`, `daycare`, `training`, `addon`, `retail` |
| `base_price` | number | — | Price in dollars |
| `price_type` | string | — | `nightly`, `fixed`, `tiered` |
| `price_unit` | string | — | `per_night`, `per_mile`, `fixed` |
| `is_active` | boolean | — | Available for booking (default `true`) |
| `is_package` | boolean | — | Whether this is a package with sub-services |

## Example

```bash
curl -X POST https://mex-pet-app.vercel.app/api/v1/services \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Grooming Package",
    "category": "grooming",
    "base_price": 75,
    "price_type": "fixed",
    "price_unit": "fixed",
    "is_active": true,
    "is_package": true
  }'
```

## Response `201`

```json
{
  "id": "svc_gg7b8c",
  "object": "service",
  "title": "Grooming Package",
  "name": "Grooming Package",
  "description": null,
  "category": "grooming",
  "base_price": 75,
  "price_type": "fixed",
  "price_unit": "fixed",
  "is_active": true,
  "is_package": true,
  "created_at": "2026-07-09T17:00:00.000Z"
}
```

After creating a package, add its items via [`POST /sub-services`](/api-reference/sub-services/create-a-sub-service) using this service's `id` as the `service_id`.
