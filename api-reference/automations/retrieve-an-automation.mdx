---
title: "Retrieve an automation"
description: "Retrieves a single automation workflow by ID. Requires the automations.read scope."
openapi: "GET /automations/{id}"
---

Retrieves a single automation by ID. Use this to check the current status of a workflow before activating or pausing it.

**Required scope:** `automations.read`

## Example

```bash
curl https://mex-pet-app.vercel.app/api/v1/automations/auto_mm3h4i \
  -H "Authorization: Bearer mxp_live_your_token_here"
```

```json
{
  "id": "auto_mm3h4i",
  "object": "automation",
  "name": "New Contact Welcome SMS",
  "status": "active",
  "trigger_type": "contact.created",
  "folder_id": null,
  "created_at": "2026-03-01T00:00:00.000Z",
  "updated_at": "2026-06-15T10:00:00.000Z"
}
```
