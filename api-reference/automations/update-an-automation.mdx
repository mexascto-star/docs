---
title: "Activate / pause an automation"
description: "Set the automation status to active, paused, or draft. Requires the automations.write scope."
openapi: "PATCH /automations/{id}"
---

Changes the status of an automation or renames it. This is how you turn automations on and off programmatically — for example, pausing a welcome SMS sequence during a maintenance window.

**Required scope:** `automations.write`

**Base URL:** `https://mex-pet-app.vercel.app/api/v1`

## Request

```bash
PATCH /automations/{id}
Authorization: Bearer mxp_live_your_token_here
Content-Type: application/json
```

## Body fields

| Field | Type | Description |
| --- | --- | --- |
| `status` | string | `active`, `paused`, or `draft` |
| `name` | string | Rename the automation |

## Status transitions

| From → To | Effect |
| --- | --- |
| `draft` → `active` | Begins firing on trigger events |
| `active` → `paused` | Trigger events ignored until re-activated |
| `paused` → `active` | Resumes firing |
| Any → `draft` | Returns to draft — use before editing in the app |

## Example — pause

```bash
curl -X PATCH https://mex-pet-app.vercel.app/api/v1/automations/auto_mm3h4i \
  -H "Authorization: Bearer mxp_live_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{ "status": "paused" }'
```
